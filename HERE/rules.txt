Rules for HTML that imports exactly
1. Always declare box-sizing explicitly
Tailwind forces box-sizing: border-box on everything. If your layout depends on content-box (browser default), elements will be the wrong size.


/* Put this at the top of your <style> block */
*, *::before, *::after { box-sizing: border-box; }
Pick one and stick to it on every element. Never mix.

2. Always set font sizes explicitly — never rely on browser defaults
The canvas strips h1–h6 default sizes (we inject a reset, but it can still be overridden). Always declare them yourself:


/* Do this */
h1 { font-size: 48px; font-weight: 800; margin: 0 0 16px; }
h2 { font-size: 32px; font-weight: 700; margin: 0 0 12px; }
p  { font-size: 16px; line-height: 1.6; margin: 0 0 12px; }

/* Never rely on this */
h1 { color: red; } /* expects browser to set the size — it won't */
3. Never use vw, vh, or 100% on the root — use px or % inside containers
The canvas artboard is a fixed-width div (1200px / 834px / 390px). Viewport units (vw, vh) measure the full browser window, not the artboard.


/* Bad — measures the whole browser window */
.hero { width: 100vw; height: 100vh; }

/* Good — fills the artboard width */
.hero { width: 100%; min-height: 600px; }
4. Never use position: fixed
Fixed elements position relative to the browser viewport, not the canvas artboard. They will float outside the page at wrong coordinates.


/* Bad */
.navbar { position: fixed; top: 0; }

/* Good */
.navbar { position: sticky; top: 0; z-index: 100; }
/* or */
.navbar { position: relative; }
5. All images must be absolute URLs
The canvas cannot resolve relative file paths. Local images like ./images/hero.jpg will be broken.


<!-- Bad -->
<img src="./images/hero.jpg" />
<img src="../assets/photo.png" />

<!-- Good -->
<img src="https://images.unsplash.com/photo-xxxx" />
<img src="https://placehold.co/800x400" />
6. All CSS must be internal — in a <style> tag
The importer captures <style> tags and <link rel="stylesheet"> pointing to CDN URLs. Local .css files cannot be loaded.


<!-- Good: internal style -->
<head>
  <style>
    .hero { background: #6366f1; }
  </style>
</head>

<!-- Good: CDN link (Bootstrap, Tailwind, etc.) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5/dist/css/bootstrap.min.css" />

<!-- Bad: local file — will not load -->
<link rel="stylesheet" href="./styles.css" />
7. Use web-safe fonts or Google Fonts via <link>
The canvas renders inside the editor. Custom fonts must be loaded via a URL.


<!-- Good -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap" rel="stylesheet">
<style>
  body { font-family: 'Inter', sans-serif; }
</style>

<!-- Also fine — web-safe fallbacks -->
<style>
  body { font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif; }
</style>
8. Always set margin: 0; padding: 0; on body
The canvas artboard has its own layout. Body margins create unexpected white space inside it.


*, *::before, *::after { box-sizing: border-box; }
body { margin: 0; padding: 0; }
9. Use a wrapper .container — not body — for max-width centering

/* Good */
.container { max-width: 1140px; margin: 0 auto; padding: 0 24px; }

/* Bad — body margin behaves oddly inside the artboard */
body { max-width: 1140px; margin: 0 auto; }
10. Declare all colors and spacing explicitly — no CSS variables without declaring them
If you use var(--primary) you must declare it in :root {}. Missing variables silently produce nothing.


/* Must declare before using */
:root {
  --primary: #6366f1;
  --gray: #6b7280;
}
.btn { background: var(--primary); }
Quick checklist template
Every HTML file for import should start like this:


<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <!-- Google Fonts (if needed) -->
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700;800&display=swap" rel="stylesheet">
  <style>
    /* 1. Reset */
    *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }

    /* 2. Base — explicit everything */
    :root {
      --primary: #6366f1;
      --text: #1f2937;
    }
    body {
      font-family: 'Inter', sans-serif;
      font-size: 16px;
      color: var(--text);
      background: #fff;
      line-height: 1.6;
    }

    /* 3. No vw/vh on sections — use % or px */
    .section { padding: 80px 0; width: 100%; }

    /* 4. Container for centering */
    .container { max-width: 1140px; margin: 0 auto; padding: 0 24px; }

    /* 5. All fonts explicit */
    h1 { font-size: 48px; font-weight: 800; line-height: 1.1; margin: 0 0 20px; }
    h2 { font-size: 36px; font-weight: 700; line-height: 1.15; margin: 0 0 16px; }
    p  { font-size: 16px; line-height: 1.65; margin: 0 0 14px; }

    /* 6. Images */
    img { max-width: 100%; display: block; }

    /* your styles below */
  </style>
</head>
<body>
  <!-- no position:fixed, no vw/vh, no relative image paths -->
</body>
</html>
Follow these 10 rules and the import will look identical to what you see when you open the file in a browser.
